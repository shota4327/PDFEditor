## 3. Detailed Defect Analysis: Non-PDF Upload Error Suppression

### Location of Defect
- **File**: `src/components/DropZone.tsx`
- **Lines**: 24–33

### Code Snippet (`src/components/DropZone.tsx`)
```typescript
24:  const handleFiles = (fileList: FileList | File[]) => {
25:    const files = Array.from(fileList).filter(
26:      (file) => file.type === 'application/pdf' || file.name.toLowerCase().endsWith('.pdf')
27:    );
28:
29:    if (files.length > 0) {
30:      if (onFilesSelected) onFilesSelected(files);
31:      if (onFilesAdded) onFilesAdded(files);
32:    }
33:  };
```

### Root Cause Logic Chain
1. When a user selects or drops a non-PDF file (e.g. `invalid-file.txt`), `fileList` contains 1 file with `type = 'text/plain'` and `name = 'invalid-file.txt'`.
2. `Array.from(fileList).filter(...)` filters out `invalid-file.txt`, resulting in `files = []` (`files.length === 0`).
3. Line 29 checks `if (files.length > 0)`. Because `files.length` is `0`, the `if` block evaluates to `false`.
4. `onFilesSelected` callback is **NEVER** called with `[]`.
5. In `src/App.tsx`, `handleFilesSelected` contains the error handling logic:
   ```typescript
   const handleFilesSelected = async (files: File[]) => {
     if (files.length === 0) {
       setErrorMessage('無効なファイル形式です。PDFファイルをアップロードしてください。');
       return;
     }
   ```
6. Because `onFilesSelected` was never invoked, `handleFilesSelected` in `App.tsx` never runs.
7. Consequently, `setErrorMessage` is never called, the `errorMessage` state remains `null`, and the UI error element `[data-testid="error-message"]` is never rendered.
8. As a result, Playwright E2E test `T2.5: Non-PDF file upload shows error message and rejects load` fails to see the error message, and end users receive zero visual feedback when attempting to upload invalid files.

---

## 4. Remediation Recommendation (For Developer Team)

To fix the defect in `src/components/DropZone.tsx`, `handleFiles` should pass the filtered result (even if empty `[]`) or explicitly trigger the `onFilesSelected` callback when invalid files are provided:

```typescript
const handleFiles = (fileList: FileList | File[]) => {
  const files = Array.from(fileList).filter(
    (file) => file.type === 'application/pdf' || file.name.toLowerCase().endsWith('.pdf')
  );

  if (onFilesSelected) {
    onFilesSelected(files);
  }
  if (files.length > 0 && onFilesAdded) {
    onFilesAdded(files);
  }
};
```
This ensures `App.tsx` receives `files = []`, sets `errorMessage`, and renders the UI alert banner as expected.
